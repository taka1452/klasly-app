# Klasly — Impeccable Project Config

## Design Context

### Users
**Primary: Studio owners/managers** running small yoga, fitness, or dance studios (typically 30–50 members). Not a tech company, not a chain — often a solo operator or tiny team. They reach for this app between classes, in the studio lobby, or at home in the evening. They are hurried, not design-savvy, and trust the product with member data, bookings, and money.

**Secondary: Members** who book classes and buy passes from the member portal — wide age range (studios skew older in yoga, younger in dance), used on mobile more than desktop.

**Tertiary: Instructors** who check their schedule and session list.

**Job to be done:** "Keep my studio running smoothly without thinking about the software." Speed, trust, and zero confusion matter far more than novelty.

### Brand Personality
**Three words:** *Mindful. Forgiving. Approachable.*

- **Mindful** — careful, quiet, unhurried. The interface never shouts. Space is generous. Motion is restrained. Echoes the physical spaces these studios inhabit.
- **Forgiving** — soft-edged in tone. Copy never blames. Errors suggest rather than scold. Defaults are gentle. An owner at 9am on a busy Saturday should feel held, not tested.
- **Approachable** — speaks in plain language, no SaaS jargon, no clever wordplay. A studio owner who has never used a CRM should feel at home.

**Emotional target:** the user closes the tab feeling *lighter*, not wired. Confident that the right thing happened.

### Aesthetic Direction
**Direction:** refined minimalism with a quiet, editorial sensibility. Think *Japanese stationery store* or *a well-designed analog planner*, not *modern fintech dashboard*.

- **Theme:** system-aware (`prefers-color-scheme`), with a calm light palette as the primary identity. Dark mode should feel like dusk, not a trading terminal.
- **Color:** the existing brand blue (`#0074c5` family) is retained but should be used as a *rare accent*, not the dominant hue. Neutrals tinted subtly toward the brand blue. Warm, quiet surfaces over cool clinical ones.
- **Typography:** currently Inter — should be replaced with a more intentional pairing. A distinctive body face with a subtle humanist quality, plus a restrained sans or quiet display face for headings. No monospace-as-shorthand-for-technical.
- **Space & rhythm:** generous whitespace, varied spacing for hierarchy, asymmetric compositions allowed. Break the card-everywhere pattern.
- **Motion:** understated. Smooth decelerations, never bounces. One well-choreographed page-load reveal > scattered micro-interactions.

**Anti-references (what this should NOT look like):**
- Stripe Dashboard, Linear, Vercel — too cold, too tech, too urgent.
- Mindbody, ClassPass admin — busy, cluttered, CRM-brown.
- Any interface with purple→blue gradients, hero metric cards, glowing accents, or glassmorphism.

### Design Principles
1. **Quiet over loud.** Every element earns its visual weight. When in doubt, remove.
2. **Trust by restraint.** Money, bookings, and member data appear in clear, unembellished type. Decoration never touches them.
3. **Forgiving copy.** Error states and empty states speak like a helpful colleague, not a system. Never "invalid", always a gentle suggestion.
4. **Rhythm, not uniformity.** Vary spacing deliberately to create hierarchy. Resist the grid-of-identical-cards reflex.
5. **Distinctive, not decorative.** One intentional type pairing and one restrained accent color will do more than any gradient, glow, or glassmorphic surface.

### Constraints
- **Tech:** Next.js 14 (App Router), Tailwind CSS 3.4, Supabase, Stripe, Lucide icons. No component library (custom components only).
- **Accessibility:** WCAG 2.1 AA target. 44×44px minimum touch targets already in place. Must remain keyboard-accessible.
- **i18n:** English + Japanese (existing). Type choices and spacing must accommodate CJK metrics.
- **Scope for change (per owner):** *small changes OK.* Token renames, spacing pass, font swap, accent rebalancing — fine. No wholesale redesigns without approval.
